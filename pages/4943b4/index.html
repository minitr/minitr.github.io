第一种做法，按照拓扑排序的经典套路，直接完成

914. 卡牌分组

hash的部分不难，主要记录下辗转相除法计算余数

证明：令a % b = r, 所以a = k * b + r, 所以r = a - k * b,假设d为a，b的一个公约数，那么 d|a， d|b,（d|a的意思就是d整除a，也就是a能被d整除），所以a - k * b 也一定能被d整除，即 d|r， 也就是 d|(a % b)， 因此d也是b 和 (a % b)的公约数，因此a,b 的公约数和b, (a%b)的公约数也是一样的，其最大公约数也一定相同，所以gcd(a, b) = gcd(b, a % b)，终止条件为b=0，返回a即可

1497. 检查数组对是否可以被 k 整除

遇到是否整除问题，优先考虑取模，对于两个数来说，相加是否能被k整除，其实就是考虑分别与k的余数相加是否等于k，考虑到负数存在，首先以-1，k=3为例 -1%3=-1，那么-1只有加上%3等于1的数例如1或者4，才能满足相加被k整除，那么-1在hash表中就要放在2的位置，也就是(-1+3)%3的情况

1146. 快照数组

最直接的想法，就是获取设定二维的vector arr，每次快照都新开一个vector放进arr中，但是这样由于快照功能snap() 的调用次数可能很多，所以我们如果采用每次快照都整体保存一次数组的方法，无论在时间复杂度还是空间复杂度上，都是行不通的。那么更优化的方法就是，只保存每次快照变化的部分！所以思路如下：

  1. 仍然为二维的vector，但是存储的是位置，只要该位置的数值被set了，再放入新值，而不是每次快照都放入一个新的vector
  2. 在指定位置下标的vector中，利用二分查找，找到get的值，也就是快照编号等于指定的值的最右边的位置

1498. 满足条件的子序列数目

思路自然是先排序，利用双指针，查找边界，在边界内的元素只有取或不取两种情况，所以应该是2^k次幂，k为长度，求次幂过程偷懒使用pow

但是仍然超过范围，这里就需要在得到次幂的过程中就要取模了，这里关于快速幂有个公式需要用到，对于乘法和加法

(a*b)%m=((a%m)*(b%m))%k;

(a+b) % k = (a % k + b % k) %k

并且修改查找边界为二分查找
